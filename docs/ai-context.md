# AI-контекст: Rules та Skills

Проєкт супроводжується машиночитними правилами для AI-асистентів, щоб згенерований
код відповідав архітектурі та стилю.

## Rules (правила)

| Файл | Інструмент | Призначення |
| --- | --- | --- |
| `CLAUDE.md` | Claude Code | Контекст проєкту, архітектурні та стильові правила, правила тестування. Завантажується автоматично в кожній сесії. |
| `.cursor/rules/go.mdc` | Cursor | Ті самі правила у форматі Cursor (`alwaysApply` для `**/*.go`). |
| `docs/patterns.md` | люди + AI | Канонічний опис патернів і SOLID, на який посилаються правила. |

Правила навмисно дублюють ключові обмеження в обох форматах, щоб різні асистенти
давали узгоджений результат: інверсія залежностей, вузькі інтерфейси, реєстрація
стратегій/спостерігачів замість `switch`, побудова курсів лише через `Factory`,
табличні тести й поріг покриття ≥ 70%.

## Skills (навички)

- **Graphify knowledge graph** — для навігації кодовою базою без зчитування всіх
  файлів. Якщо існує `graphify-out/graph.json`, асистент спершу запитує граф
  (`graphify query/explain/path`), і лише потім читає сирці. Після зміни коду —
  `graphify update .`, щоб індекс лишався актуальним.

## Як це оцінюється
Наявність якісних, специфічних для проєкту правил (а не загальних порад) і їх
відповідність реальній архітектурі — частина критерію «Архітектура, Патерни та
ШІ-контекст».
